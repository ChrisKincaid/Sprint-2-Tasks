------------- Task Notes are below this line -------------

  • Initial Setup:
    • Create a new GitHub repository named "JS-Practical-Components".
    • Clone the repository to your local machine.

    • I am doing all of Sprint 2 in one repository and making a branch for each task.

  • Engage with Video Lessons:
    • Watch the videos from the provided playlist that covers:
      • DOM & DOM Events
      • Building a Pricing Calculator
      • Styling and Interacting with Buttons
      • Creating a Coding Playground
      • Designing a Stopwatch
      • Implementing an Accordion
      • Crafting a Scrolling Progress Indicator
    • For each video segment, jot down key points, techniques applied, and any challenges the instructor addresses.
  • Implementation of Components:
    • Pricing Calculator: Build a simple pricing calculator based on user inputs.
    • Button Playground: Create various styled buttons with different interactions.
    • Coding Playground: Implement a small area where users can write and test out JavaScript.
    • Stopwatch: Create a functional stopwatch with start, pause, and reset functionalities.
    • Accordion: Implement an accordion for FAQs or similar content.
    • Scrolling Progress: Build a progress bar that fills up as a user scrolls down a webpage.
  • Commit Practices:
    • After building each component, commit the changes with a descriptive message, e.g., "Implemented the Pricing Calculator".
    • Once you've finished all components and ensured they function correctly, push the commits to your GitHub repository.
  • Documentation & Analysis:
    • Update the repository's README.md file with:
      • Project Overviews: A brief description of each component, the techniques applied, and any challenges faced.
      • Screenshots: If possible, attach screenshots or GIFs of each functioning component.
  • Reflection:
    • At the end of the task, add a brief retrospective to the README.md about your experience. Reflect on:
      • The most enlightening part of the video content.
      • The complexity of implementing each component.
      • How these practical tasks might affect your approach to JavaScript in future projects.
